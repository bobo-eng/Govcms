# Progress: GovCMS

## 当前总体状态

截至 **2026-03-11**，项目状态已经从“后台 MVP 原型收口”切换为“正式项目分层与口径重构完成，进入标准版建设准备阶段”。

当前统一状态如下：

| 层级 / 阶段 | 状态 |
| --- | --- |
| L0 当前基座 | 已交付 |
| M1 文档基线与统一口径 | 已完成 |
| M2 CMS 平台共享层 | 规划中 |
| M3 后台标准版 | 规划中 |
| M4 门户标准版 | 规划中 |
| M5 信创与国密正式交付 | 规划中 |

## 本次完成事项（2026-03-11）

- 已完成项目重新分层，统一为 `L0-L4` 五层口径。
- 已明确当前仓库只属于 `L0 当前基座`，不再混写为全量方案已完成。
- 已在 `docs/` 下建立新的主文档体系，作为唯一事实来源。
- 已将信创适配与国密要求提升为正式交付基线。
- 已明确门户统一定义：由 CMS 建站并静态化发布，对外以静态站点为主运行。

## 当前已确认的 L0 能力

- 登录认证与 JWT 鉴权
- RBAC 权限模型
- 动态菜单与按钮级权限控制
- 仪表盘
- 用户、角色、权限、菜单管理
- 内容基础管理
- 站点基础管理
- 媒体基础管理
- 本地文件存储与基础测试链路

## 当前主要缺口

- 缺少站点建站核心对象：栏目、专题、模板、导航、发布作业
- 缺少后台标准版关键链路：审核中心、发布中心、审计日志
- 缺少门户标准版：首页、栏目页、详情页、专题页、搜索、静态发布
- 缺少正式交付所需的信创适配与国密落地

## 下一步重点

1. 进入 `M2 CMS 平台共享层` 设计与实现
2. 优先定义栏目管理、模板管理、内容生命周期与发布契约
3. 为 KingbaseES、TongWeb、国密能力建立基础抽象
4. 再进入后台标准版与门户标准版编码阶段
