# Progress: GovCMS

## 当前总体状态

截至 **2026-03-17**，项目整体已从“后台 MVP 原型”推进到“内容生命周期 + 审核工作区 + 发布中心 MVP”阶段。当前统一判断如下：

| 层级 / 阶段 | 状态 |
| --- | --- |
| L0 当前基座 | 已交付 |
| M1 文档基线与统一口径 | 已完成 |
| M2 CMS 平台共享层 | 进行中（栏目 / 模板 / 生命周期 / 发布契约首批已落地） |
| M3 后台标准版 | 进行中（内容中心、审核工作区、发布中心 MVP 已落地） |
| M4 门户标准版 | 规划中 |
| M5 信创与国密正式交付 | 规划中 |

## 最近已完成事项（截至 2026-03-17）

- 已完成栏目管理后端主链路：树、CRUD、移动、排序、影响范围、模板关联校验。
- 已完成模板管理后端主链路：模板创建、版本保存、版本回滚、绑定、解绑、影响范围与预览。
- 已完成内容生命周期六态模型：`draft / pending_review / rejected / approved / published / offline`。
- 已完成审核工作区首批能力：查看待审核内容、审核通过、驳回。
- 已完成发布中心 MVP：发布前校验、影响范围、同步执行、下线、回滚、产物与日志记录。
- 已引入 `RenderContextAssembler` 与 `PortalRenderService` 正式用于发布链路的受控页面渲染。
- 已完成菜单权限收口与角色可见菜单隔离。
- 已完成站点 options 只读接口，解除内容页 / 审核页 / 发布页对站点管理接口的错误依赖。
- 已完成角色体系定义稿与导航 / 专题 / 站点管理员建模稿初版。

## 当前主要缺口

- 缺少独立导航对象与导航管理工作台
- 缺少独立专题对象、专题模板与专题页正式入口
- 缺少站点管理员正式角色实现
- 缺少 `topic-page` 正式渲染与发布支持
- 缺少异步发布编排、搜索索引与整站标准版增强能力
- 缺少正式交付所需的信创适配与国密落地

## 下一步重点

1. 先完成角色体系产品定稿。
2. 再完成导航 / 专题 / 站点管理员建模定稿。
3. 在对象和角色边界明确后，再进入站点治理、导航管理、专题管理的实现。
4. 最后推进门户标准版增强、信创适配与国密交付。
