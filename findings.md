# Findings: 政府CMS 项目研究

## 技术选型

### 后端
| 技术 | 选择 | 理由 |
|------|------|------|
| 框架 | Spring Boot 3.x | 最新LTS，支持Java 17+ |
| 安全 | Spring Security + JWT | 成熟方案 |
| 工作流 | Flowable | 开源BPM引擎，支持复杂审批 |
| ORM | MyBatis-Plus | 国产数据库兼容性最好 |
| 缓存 | Redis | 会话与数据缓存 |

| 技术 | 选择 | 理由 |
|------|------|------|
| 框架 | Vue 3 | Composition API |
| UI | Ant Design Vue | Vue3官方推荐 |
| 状态 | Pinia | Vue3官方推荐 |
| 构建 | Vite | 快速开发 |

### 数据库
| 用途 | 数据库 | 理由 |
|------|--------|------|

| 国产 | 达梦DM8 | 政府项目常用 |
| 国产 | 金仓KingbaseES | 政府项目常用 |

## 架构决策

### 1. 单应用多组织架构
- 共享代码库，不同组织数据隔离
- 组织ID作为所有业务表的外键
- 优点: 维护成本低，部署简单

### 2. 数据隔离方案
- 逻辑隔离 (非物理隔离)
- 通过租户拦截器实现
- 组织级别: 用户只能访问所属组织数据

### 3. 审核工作流
- 使用 Flowable 引擎
- 可视化流程设计
- 支持会签、加签、驳回

## 已识别的风险

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| 国产数据库兼容性 | 高 | 使用 MyBatis-Plus，其对国产数据库支持较好 |
| 多组织数据隔离 | 高 | 租户拦截器 + 数据权限注解 |
| 大文件上传 | 中 | MinIO 对象存储 |
| 并发性能 | 中 | Redis 缓存 + 读写分离 |
