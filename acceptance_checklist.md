# GovCMS MVP 验收清单

## 1. 环境前置条件
- MySQL 已启动，且已创建数据库 `govcms`
- `src/main/resources/application.yml` 默认 Profile 为 `local`
- 后端端口为 `8080`，前端端口为 `4869`
- 媒体文件目录默认为 `./storage/media`

## 2. 启动检查

### 2.1 启动后端
操作步骤：
1. 在项目根目录执行 `.\.tools\apache-maven-3.9.12\bin\mvn.cmd -Dmaven.repo.local=.\.m2\repository spring-boot:run`
2. 打开 `http://127.0.0.1:8080`

预期结果：
- 后端正常启动
- 未登录访问业务接口时返回 `401` 或 `403`

### 2.2 启动前端
操作步骤：
1. 在 `frontend` 目录执行 `npm install`
2. 执行 `npm run dev`
3. 打开 `http://127.0.0.1:4869`

预期结果：
- 登录页正常打开

### 2.3 默认管理员登录
操作步骤：
1. 使用 `admin / admin123` 登录
2. 如浏览器存在旧会话，先重新登录一次

预期结果：
- 登录成功并进入仪表盘
- 左侧菜单包含用户、角色、权限、菜单、内容、站点、媒体入口

## 3. 核心功能验收

### 3.1 仪表盘
操作步骤：打开仪表盘页面，查看统计卡片。

预期结果：
- 仪表盘正常加载
- 站点数为数据库真实数量

### 3.2 用户管理
操作步骤：打开用户管理页面，查看列表，尝试新增、编辑、删除、重置密码。

预期结果：
- 页面正常工作
- 对应按钮和接口权限一致

### 3.3 角色管理
操作步骤：打开角色管理页面，尝试新增、编辑、删除、分配权限。

预期结果：
- 页面正常工作
- 角色权限更新后重新登录可见效果

### 3.4 权限管理
操作步骤：打开权限管理页面，查看权限树并尝试增删改。

预期结果：
- 页面正常加载
- 无权限按钮不会显示

### 3.5 菜单管理
操作步骤：打开菜单管理页面，查看列表并尝试增删改。

预期结果：
- 菜单管理正常工作
- 导航展示与菜单配置一致

### 3.6 内容管理
操作步骤：打开内容管理页面，尝试新增、编辑、发布 / 下线、删除文章。

预期结果：
- 内容页面正常工作
- 发布 / 下线动作受权限控制

### 3.7 站点管理
操作步骤：
1. 打开站点管理页面
2. 进行关键字 / 状态 / 组织 ID 筛选
3. 新建站点
4. 编辑站点
5. 执行启用 / 禁用
6. 删除站点

预期结果：
- 页面标题显示“站点管理”
- 列表、筛选、新增、编辑、启停、删除均可正常完成

### 3.8 媒体管理
操作步骤：
1. 打开媒体管理页面
2. 上传一张图片或一个 PDF / 文档文件
3. 使用关键字和类型筛选
4. 点击“预览”或“打开”
5. 删除该文件

预期结果：
- 页面标题显示“媒体管理”
- 上传成功后列表立即出现文件
- 图片和 PDF 可预览；其他文档可正常打开 / 下载
- 删除成功后列表不再显示该文件

## 4. 边界场景
- 上传超过 `20MB` 的文件，应返回失败提示
- 上传不支持扩展名的文件，应返回失败提示
- 上传同名文件两次，不应覆盖旧文件
- 删除媒体记录后再次访问预览接口，应返回 `404`
