# GovCMS 栏目管理 PRD

## 1. 文档信息

- 文档名称：栏目管理 PRD
- 所属阶段：`L1 CMS 平台共享层` / `L2 后台标准版`
- 当前状态：V1.0 初稿
- 最近更新：2026-03-11
- 适用范围：CMS 后台建站、内容归属、门户栏目页生成、静态化发布依赖计算

## 2. 产品定位

栏目管理是 GovCMS 建站能力的核心模块，用于定义站点的信息架构、内容归属关系、门户导航结构和页面生成基础。

栏目不是单纯的“分类标签”，而是连接以下能力的中枢对象：

- 站点结构
- 内容归属
- 导航展示
- 模板绑定
- URL 路由
- 发布影响范围
- 门户栏目页生成

栏目管理需要服务于“CMS 建站 + 门户静态化发布”模式，因此其设计必须同时满足后台运营效率和前台门户生成稳定性。

## 3. 产品目标

### 3.1 目标

- 支持站点管理员快速搭建清晰、稳定的栏目树
- 支持内容按栏目归属、聚合、展示和发布
- 支持栏目与模板、导航、发布规则建立稳定绑定关系
- 支持门户栏目页、面包屑、导航和聚合位生成
- 为后续模板管理、发布中心和门户静态化提供稳定输入

### 3.2 非目标

首期栏目管理不负责以下能力：

- 自由拖拽页面搭建器
- 多站点之间栏目复制编排工作流
- 栏目级复杂审批流
- 栏目级个性化 AB 测试
- AI 自动生成栏目结构

## 4. 适用角色

| 角色 | 使用目标 |
| --- | --- |
| 平台管理员 | 查看全局栏目结构、管理权限边界、排查异常 |
| 站点管理员 | 创建栏目树、配置模板、绑定导航、调整排序 |
| 编辑 | 选择内容归属栏目、查看栏目状态与模板限制 |
| 审核人员 | 审核内容时确认栏目归属是否正确 |
| 运维人员 | 发布时查看栏目变更影响范围 |

## 5. 业务边界

栏目管理与其他模块的关系如下：

- 与 `Site`：栏目必须归属于某个站点
- 与 `Content`：内容必须至少归属一个主栏目
- 与 `Template`：栏目需要绑定栏目页模板，必要时可绑定详情页模板规则
- 与 `Navigation`：栏目可配置是否进入站点导航
- 与 `PublishJob`：栏目变更会影响栏目页、导航、面包屑、聚合位和部分详情页
- 与 `Portal`：门户栏目页由栏目配置和栏目模板生成

## 6. 核心场景

### 6.1 新站建站

站点管理员新建站点后，首先搭建栏目树，明确一级栏目、二级栏目和内容归属逻辑，再绑定模板和导航规则。

### 6.2 栏目结构调整

站点运营过程中，管理员需要新增栏目、调整父子关系、修改排序、停用栏目或下线栏目展示。

### 6.3 内容归属与发布

编辑创建内容时，必须选择归属栏目；发布系统根据栏目决定详情页入口、栏目页聚合位置和面包屑路径。

### 6.4 门户导航展示

门户前台的导航、栏目页入口、列表页和面包屑均来自栏目结构与栏目显示配置。

### 6.5 发布影响计算

当栏目名称、路径、模板、排序、父级关系发生变化时，系统需要识别受影响页面，并纳入整站或增量发布范围。

## 7. 核心设计原则

- 栏目必须归属于站点，不允许脱离站点独立存在
- 栏目树必须稳定，避免频繁破坏 URL 和门户结构
- 栏目配置优先服务门户生成，而不是服务后台展示本身
- 栏目删除要严格受控，优先防止误删造成内容丢失或页面失效
- 首期设计以政务门户常见信息架构为基准，强调清晰、可控、可追溯

## 8. 功能范围

### 8.1 栏目树视图

支持站点维度的栏目树展示，默认按树形展示全部栏目。

功能要求：

- 支持按站点切换查看栏目树
- 支持展开/折叠栏目节点
- 支持显示栏目名称、编码、层级、排序、状态、模板绑定状态、内容数量
- 支持按名称/编码搜索栏目
- 支持定位栏目所属父级路径

### 8.2 新增栏目

支持新增顶级栏目和子栏目。

新增时需配置：

- 所属站点
- 父级栏目
- 栏目名称
- 栏目编码
- 栏目类型
- 栏目路径段
- 排序
- 栏目描述
- 导航显示
- 栏目状态
- 列表模板绑定
- 详情模板规则（可选）
- 内容聚合方式
- SEO 信息（标题、关键词、描述）

### 8.3 编辑栏目

支持修改栏目基础信息、模板绑定、展示规则和导航规则。

编辑时需重点关注：

- 改名是否影响导航与面包屑
- 改路径是否影响门户 URL 和历史链接
- 改模板是否影响栏目页渲染结果
- 改父级是否影响整条栏目路径

### 8.4 栏目排序与层级调整

支持在同级栏目下排序调整，并支持有限的父子层级调整。

首期要求：

- 支持同级排序修改
- 支持将栏目移动到新的父级栏目下
- 移动前需提示受影响范围
- 不支持跨站点移动栏目

### 8.5 栏目模板绑定

栏目需要支持与模板管理模块联动。

首期要求：

- 每个栏目必须绑定一个栏目页模板
- 可按栏目类型或栏目节点绑定默认详情模板规则
- 模板变更需纳入发布影响范围
- 模板失效时栏目不可进入正式发布

### 8.6 栏目导航规则

栏目可配置是否在门户导航中显示。

首期要求：

- 支持顶部导航显示/不显示
- 支持导航排序
- 支持面包屑参与/不参与
- 支持是否对外开放访问

### 8.7 栏目内容聚合规则

栏目作为列表页，需要支持内容聚合。

首期仅支持两种聚合方式：

- 手工归属：仅展示明确归属到该栏目的内容
- 自动聚合：展示该栏目及其子栏目下的已发布内容

首期不支持复杂条件组合、标签规则引擎、跨站点聚合。

### 8.8 栏目删除与停用

栏目删除需要严格控制。

首期规则：

- 存在子栏目时不可删除
- 存在已归属内容时不可删除
- 已绑定导航且仍生效时不可删除
- 已被发布产物引用时需先完成下线或调整
- 删除操作必须二次确认

停用规则：

- 停用后栏目不可继续接收新发布内容
- 停用后默认不在导航展示
- 停用栏目是否继续保留历史页面，由发布中心策略决定

## 9. 数据模型建议

建议核心字段如下：

| 字段 | 说明 | 是否必填 |

| parentId | 父级栏目 ID，顶级为空 | 否 |
| name | 栏目名称 | 是 |
| code | 栏目编码，站点内唯一 | 是 |
| type | 栏目类型 | 是 |
| slug | URL 路径段 | 是 |
| fullPath | 栏目完整路径 | 是 |
| sort | 同级排序 | 是 |
| status | 栏目状态 | 是 |
| navVisible | 是否显示在导航 | 是 |
| breadcrumbVisible | 是否参与面包屑 | 是 |
| listTemplateId | 栏目页模板 ID | 是 |

## 10. 业务规则

### 10.1 栏目层级规则

- 首期支持树形多级结构
- 产品建议实际运营层级不超过 4 级
- 同一站点内栏目必须形成唯一树结构，不允许循环引用
- 子栏目必须与父栏目属于同一站点

### 10.2 唯一性规则

- `code` 在同一站点内唯一
- `fullPath` 在同一站点内唯一
- 同一父级下栏目名称不建议重复，首期默认不允许重复

### 10.3 URL 规则

- 栏目路径由父级路径 + 当前栏目 `slug` 拼接生成
- `slug` 仅允许字母、数字、中划线和下划线
- 修改父栏目或 `slug` 时，需要提示 URL 变更影响

### 10.4 模板规则

- 栏目上线前必须具备有效栏目模板
- 模板未绑定或模板状态无效时，栏目不可进入发布范围
- 模板切换需纳入栏目变更发布影响范围

### 10.5 内容归属规则

- 内容必须选择一个主栏目
- 可保留后续扩展“副栏目/多栏目归属”能力，但首期不做
- 栏目停用后，不允许新增内容归属到该栏目

## 11. 状态设计

栏目首期仅采用简单状态模型：

| 状态 | 说明 |
| --- | --- |
| enabled | 启用，可被导航、内容、发布使用 |
| disabled | 停用，不接收新内容，不进入正常发布 |

说明：

- 栏目本身不引入复杂审核流
- 审核流属于内容生命周期，不属于栏目生命周期

## 12. 权限设计

建议栏目权限编码如下：

| 权限编码 | 说明 |
| --- | --- |
| `content:column:view` | 查看栏目 |
| `content:column:create` | 新增栏目 |
| `content:column:update` | 编辑栏目 |
| `content:column:delete` | 删除栏目 |
| `content:column:sort` | 调整排序与层级 |
| `content:column:template-bind` | 绑定模板 |
| `content:column:publish-impact:view` | 查看栏目发布影响 |

## 13. 页面结构建议

栏目管理页面建议采用“双栏结构”：

- 左侧：站点切换 + 栏目树 + 搜索
- 右侧：栏目详情 / 编辑表单 / 受影响范围 / 模板绑定信息

页面至少包含以下操作入口：

- 新增顶级栏目
- 新增子栏目
- 编辑栏目
- 调整排序
- 移动栏目
- 停用栏目
- 删除栏目
- 查看栏目影响范围
- 查看栏目下内容数量

## 14. 与发布中心的契约

栏目变更需要输出标准发布影响类型，首期至少包括：

- 栏目页重建
- 父级栏目页重建
- 子栏目页重建
- 导航重建
- 面包屑路径重建
- 受影响内容详情页重建
- 搜索索引重建

栏目管理模块不负责执行发布，但必须向发布中心提供清晰的变更结果。

## 15. 接口建议

首期建议接口分组如下：

- `GET /api/columns/tree`：按站点读取栏目树
- `GET /api/columns/{id}`：读取栏目详情
- `POST /api/columns`：创建栏目
- `PUT /api/columns/{id}`：更新栏目
- `PUT /api/columns/{id}/sort`：调整栏目排序
- `PUT /api/columns/{id}/move`：调整父级栏目
- `PUT /api/columns/{id}/status`：启用/停用栏目
- `DELETE /api/columns/{id}`：删除栏目
- `GET /api/columns/{id}/impact`：查看栏目变更影响范围
- `GET /api/columns/{id}/stats`：查看栏目内容数量和引用情况

## 16. 验收标准

### 16.1 功能验收

- 能按站点创建顶级栏目和子栏目
- 能展示稳定的栏目树结构
- 能编辑栏目名称、编码、路径、模板、状态和导航规则
- 能查看栏目下内容数量和模板绑定状态
- 能执行同级排序与父级调整
- 能阻止非法删除

### 16.2 业务验收

- 内容创建时可以正确选择栏目
- 栏目模板绑定后可作为门户栏目页生成输入
- 栏目变更后可计算受影响页面范围
- 停用栏目后不允许继续挂接新发布内容

### 16.3 风险控制验收

- 栏目循环引用被禁止
- 重复编码与重复路径被禁止
- 删除存在子栏目或内容的栏目时被阻止
- 关键变更存在明确二次确认和影响提示

## 17. 后续扩展位

以下能力保留到后续阶段，不纳入本期栏目管理交付：

- 栏目复制
- 栏目批量导入导出
- 栏目多模板策略编排
- 多栏目归属内容模型
- 栏目级专题聚合规则引擎
- 栏目个性化推荐与运营位配置

## 18. 当前结论

对于 L1 而言，栏目管理不是后台附属功能，而是建站能力的基础设施。

在实施顺序上，栏目管理必须与模板管理、内容生命周期和发布契约一起设计，但在产品拆分上，栏目管理应当作为第一份详细 PRD 优先落地。
