# GovCMS 发布渲染契约 PRD

## 1. 文档信息

- 文档名称：发布渲染契约 PRD
- 所属阶段：`L1 CMS 平台共享层` / `L2 后台标准版` / `L3 门户标准版`
- 当前状态：V1.0 初稿
- 最近更新：2026-03-11
- 适用范围：CMS 发布中心、静态化门户生成、增量发布、回滚、页面重建规则

## 2. 产品定位

发布渲染契约是 GovCMS 中连接“后台业务对象”和“门户静态产物”的统一规则层，用于回答以下关键问题：

- 什么对象可以触发发布
- 何时允许进入正式发布
- 发布时需要重建哪些页面
- 门户最终消费什么数据与模板组合
- 失败后如何重试与回滚

它不是单独的 UI 模块，而是栏目管理、模板管理、内容生命周期、门户静态化发布之间的共同协议。

在 GovCMS 体系中，发布渲染契约承担以下职责：

- 统一发布输入对象定义
- 统一整站发布与增量发布规则
- 统一渲染上下文和模板渲染边界
- 统一页面重建影响范围计算
- 统一发布产物、日志、失败重试与回滚约束

## 3. 产品目标

### 3.1 目标

- 建立统一、可执行、可审计的发布规则体系
- 明确哪些业务变更会触发哪些门户页面重建
- 保证后台对象与门户静态产物之间的映射稳定
- 支持整站发布和增量发布两种模式
- 为门户标准版编码实现提供清晰契约，避免边做边猜

### 3.2 非目标

首期发布渲染契约不负责以下能力：

- 多地域多集群分发编排
- 灰度发布、蓝绿发布与流量切换
- 页面级实时流式渲染
- 自定义脚本执行型模板渲染器
- 分布式发布编排平台的复杂调度能力

## 4. 适用角色

| 角色 | 使用目标 |
| --- | --- |
| 平台管理员 | 定义发布规则边界、审计发布任务、查看异常 |
| 站点管理员 | 发起整站发布、查看影响范围、确认发布结果 |
| 编辑 | 理解内容发布后会影响哪些页面 |
| 审核人员 | 确认审核通过内容是否满足正式发布条件 |
| 运维人员 | 执行发布、失败重试、回滚和产物核查 |

## 5. 业务边界

发布渲染契约与其他模块的关系如下：

- 与 `Site`：站点是整站发布的顶层作用域
- 与 `Column`：栏目变更影响栏目页、导航、面包屑与聚合位
- 与 `Content`：内容发布和下线影响详情页、栏目页、首页推荐位、搜索索引
- 与 `Template`：模板决定页面如何渲染；模板变更影响对应页面集合
- 与 `Navigation`：导航变更影响站点级菜单、栏目入口和面包屑结果
- 与 `Portal`：门户最终只消费“已发布对象 + 有效模板 + 有效站点结构”生成的静态产物
- 与 `AuditLog`：每次发布、重试、回滚都必须可审计

## 6. 契约总览

### 6.1 发布触发对象

首期允许触发发布的对象如下：

- 站点
- 栏目
- 内容
- 专题
- 模板
- 导航配置

### 6.2 发布模式

首期固定支持两种模式：

- `full`：整站发布
- `incremental`：增量发布

说明：

- `full` 适用于新站初始化、模板大改、站点结构大改、整站重建
- `incremental` 适用于单篇内容发布、栏目局部变更、专题局部变更、导航局部调整

### 6.3 渲染输入

一次合法发布必须至少具备以下输入：

- 已发布或待发布的业务对象
- 有效站点结构
- 有效栏目结构
- 有效模板绑定
- 受控渲染上下文
- 发布目标环境信息

### 6.4 渲染输出

发布输出必须以标准产物的形式落地，至少包括：

- 首页页面产物
- 栏目页产物
- 详情页产物
- 专题页产物
- 404 页面产物
- 搜索索引产物
- 发布日志与结果记录

## 7. 核心定义

### 7.1 已发布数据边界

门户渲染只允许消费以下数据：

- `published` 状态内容
- 有效状态栏目
- 有效状态模板
- 有效站点与导航配置
- 允许对外展示的专题和聚合配置

门户不得直接消费以下数据：

- 草稿内容
- 审核中内容
- 驳回内容
- 停用栏目
- 草稿模板
- 未正式发布的专题结构

### 7.2 页面类型定义

首期门户页面类型固定如下：

- `home`：首页
- `column-list`：栏目列表页
- `content-detail`：内容详情页
- `topic-page`：专题页
- `error-404`：404 页面

### 7.3 渲染上下文定义

模板渲染时允许使用的上下文分组如下：

- `siteContext`：站点信息、域名、全局配置
- `navigationContext`：导航、面包屑、栏目入口
- `columnContext`：栏目结构、路径、聚合信息
- `contentContext`：已发布内容字段、媒体引用、SEO 字段
- `topicContext`：专题结构与专题聚合内容
- `renderMeta`：发布时间、版本、构建号、任务号

首期不允许模板直接读取未声明上下文。

## 8. 核心场景

### 8.1 单篇内容发布

内容审核通过后进入发布中心，系统计算受影响页面：

- 当前内容详情页
- 所属栏目页
- 父级栏目聚合位（如适用）
- 首页推荐位（如配置命中）
- 搜索索引

### 8.2 单篇内容下线

内容下线后，系统需重建：

- 当前内容详情页（下线或移除）
- 所属栏目页
- 首页推荐位
- 搜索索引

### 8.3 栏目结构变更

栏目名称、路径、排序、父级、模板或导航显示规则变更时，系统需识别：

- 当前栏目页
- 子栏目页
- 导航结构
- 面包屑路径
- 相关详情页的路径引用

### 8.4 模板变更

模板新版本生效或模板绑定变更时，系统需识别：

- 绑定该模板的首页、栏目页、详情页、专题页
- 使用同一公共布局的页面集合
- 受影响搜索或结构化元信息（如适用）

### 8.5 导航变更

导航顺序、可见性、栏目入口关系变化时，系统需重建：

- 首页导航区
- 栏目页导航区
- 面包屑相关页面

### 8.6 整站发布

整站发布应生成站点完整静态产物，包括：首页、栏目页、详情页、专题页、404 页、搜索索引与发布结果清单。

## 9. 核心设计原则

- 发布必须基于明确对象和明确规则执行，不允许隐式发布
- 渲染必须消费受控数据，不允许模板跨边界读取任意数据
- 增量发布优先以“影响范围可枚举”为前提
- 发布成功与业务状态变更必须保持一致性
- 失败、重试、回滚必须有完整日志与审计记录

## 10. 功能范围

### 10.1 发布前校验

发布动作执行前必须做统一校验。

首期至少校验：

- 站点状态有效
- 栏目结构完整且无非法状态
- 内容状态满足发布要求
- 模板状态有效且绑定完整
- 媒体引用资源可用
- 导航配置可解析
- 发布目标路径和环境配置可用

### 10.2 影响范围计算

系统必须能根据触发对象生成影响范围。

首期要求：

- 支持查看影响页面类型
- 支持查看预估重建页面数量
- 支持区分 `full` 与 `incremental` 模式
- 支持将影响结果记录到发布任务中

### 10.3 发布计划生成

发布中心根据触发对象和影响范围生成发布计划。

发布计划至少包含：

- 发布任务编号
- 任务类型
- 站点范围
- 触发对象
- 受影响页面集合
- 执行顺序
- 预估产物数量

### 10.4 渲染执行

渲染执行阶段需完成：

- 装载受控上下文
- 匹配页面模板
- 生成静态页面产物
- 写入产物目录
- 输出执行结果和失败信息

### 10.5 产物登记

每次发布完成后，系统需登记产物信息。

首期要求：

- 记录产物类型
- 记录产物路径
- 记录关联任务号
- 记录生成时间
- 记录版本与摘要信息

### 10.6 失败处理与重试

发布失败时，系统需支持明确处理。

首期要求：

- 标记失败任务
- 记录失败阶段与原因
- 支持任务级重试
- 支持失败任务查询

### 10.7 回滚

回滚用于恢复上一稳定发布结果。

首期要求：

- 支持按站点回滚到上一稳定版本
- 支持按任务号回滚
- 回滚必须记录来源任务和目标版本
- 回滚后必须形成新任务记录

## 12. 业务规则

### 12.1 触发规则

- 内容从 `review` 进入 `published` 才可触发正式发布
- 内容从 `published` 进入 `offline` 必须触发下线类更新
- 栏目、模板、导航的结构性变更必须重新计算影响范围
- 站点级关键配置变更默认触发整站发布

### 12.2 增量规则

- 增量发布必须能明确列出影响页面集合
- 不能准确识别影响范围时，应降级为整站发布
- 增量发布不得遗漏栏目页、详情页、首页聚合位、搜索索引等关联页面

### 12.3 一致性规则

- 只有发布任务成功，内容才能对门户正式可见
- 若渲染失败，不应错误宣告门户已更新成功
- 发布结果与任务日志必须一致，可回查、可核对

### 12.4 回滚规则

- 回滚只能基于已登记的稳定发布版本
- 回滚不覆盖历史记录，而是形成新任务
- 回滚完成后门户应恢复到目标版本产物集合

## 13. 权限设计

建议发布渲染契约相关权限编码如下：

| 权限编码 | 说明 |
| --- | --- |
| `publish:view` | 查看发布任务 |
| `publish:impact:view` | 查看影响范围 |
| `publish:execute` | 执行发布 |
| `publish:retry` | 重试失败任务 |
| `publish:rollback` | 执行回滚 |
| `publish:artifact:view` | 查看发布产物 |
| `publish:log:view` | 查看发布日志 |

## 14. 页面结构建议

发布中心页面建议至少包含以下区域：

- 发布入口区：整站发布 / 增量发布
- 影响范围预览区
- 发布计划摘要区
- 执行日志区
- 发布结果区
- 失败任务区
- 回滚入口区
- 历史任务列表区

## 15. 接口建议

首期建议接口分组如下：

- `POST /api/publish/precheck`：发布前统一校验
- `POST /api/publish/impact`：计算影响范围
- `POST /api/publish/jobs`：创建发布任务
- `POST /api/publish/jobs/{id}/execute`：执行发布任务
- `POST /api/publish/jobs/{id}/retry`：重试失败任务
- `POST /api/publish/jobs/{id}/rollback`：按任务回滚
- `GET /api/publish/jobs/{id}`：查看任务详情
- `GET /api/publish/jobs/{id}/artifacts`：查看产物列表
- `GET /api/publish/jobs/{id}/logs`：查看发布日志

## 16. 验收标准

### 16.1 功能验收

- 可对站点执行整站发布
- 可对内容、栏目、模板、专题执行增量发布
- 可提前查看影响范围和预估重建数量
- 发布完成后可查看产物清单、结果和日志
- 失败任务可重试，稳定版本可回滚

### 16.2 业务验收

- 内容发布后，详情页、栏目页、首页聚合位和搜索索引可正确更新
- 栏目变更后，导航、面包屑和栏目页可正确更新
- 模板变更后，绑定对象对应页面可正确重建
- 整站发布可生成完整门户站点

### 16.3 风险控制验收

- 影响范围无法明确时不会误走不安全增量发布
- 渲染失败不会错误标记发布成功
- 回滚过程完整留痕
- 门户不会消费未发布对象
- 发布上下文严格受控，不存在越权数据读取

## 17. 后续扩展位

以下能力保留到后续阶段，不纳入本期发布渲染契约交付：

- 蓝绿发布与灰度发布
- 分布式多节点并行渲染
- 多环境差异发布编排
- CDN 自动刷新编排
- 更复杂的搜索索引异步构建体系
- 多区域门户分发能力

## 18. 当前结论

对于 L1 而言，发布渲染契约不是发布中心 UI 的附属说明，而是栏目管理、模板管理、内容生命周期和门户静态化之间的核心桥梁。

在产品推进顺序上，它必须作为 L1 第四份详细 PRD 定稿后，才能让后续编码在“对象、状态、模板、发布”四条主线之间建立统一闭环。
