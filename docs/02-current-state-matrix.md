# GovCMS 现状对照矩阵

## 1. 当前代码概览

当前仓库已经具备以下可运行代码：

- 后端控制器：认证、仪表盘、用户、角色、权限、菜单、内容、站点、媒体
- 前端页面：登录、仪表盘、用户、角色、权限、菜单、内容、站点、媒体
- 已有测试：仪表盘、站点、媒体的控制器与服务测试

这说明当前仓库已经形成后台 MVP 基座，但还没有进入 CMS 建站平台和门户发布体系。

## 3. 技术现状与目标差距

### 当前技术现状

- 后端：`Spring Boot 3.2 + Spring Security + Spring Data JPA + JWT`
- 前端：`Vue 3 + TypeScript + Vite + Ant Design Vue`
- 数据库：`MySQL 8`
- 部署形态：本地 `spring-boot:run` 与前端 `npm run dev`
- 文件存储：本地文件系统 `./storage/media`

### 正式目标技术口径

- 数据库目标：`KingbaseES`
- 应用服务器目标：`TongWeb`
- Web 层目标：`Nginx`
- 运行环境目标：国产 OS / 政务云兼容
- 安全目标：国密传输、国密签名、国密数据保护、审计与回滚能力

## 4. 当前代码可直接复用部分

- 认证与权限框架
- 后台前端工程基础设施
- 用户、角色、权限、菜单管理能力
- 站点与媒体的基础数据结构思路
- 现有测试框架和部分验证链路

## 5. 当前代码必须升级或重构部分

- 内容模型需要从“文章 CRUD”升级为“站点内容模型”
- 站点模型需要升级为完整建站模型
- 媒体模型需要升级为媒体资产中心模型
- 发布链路需要从“无门户”升级为“后台驱动门户静态发布”
- 安全能力需要从普通 JWT 升级到满足国密要求的正式方案
- 部署能力需要从本地开发口径升级为正式信创部署口径
