# GovCMS 路线图与验收

## 1. 实施路线图

### M1 文档基线与统一口径

目标：

- 完成项目定位重定义
- 明确 L0-L4 分层
- 完成当前状态对照矩阵
- 完成后台标准版、门户标准版、信创国密交付基线文档

状态：已完成。

### M2 CMS 平台共享层

目标：

- 统一站点、栏目、内容、媒体、专题、模板、发布、审计模型
- 明确模板管理能力：模板类型、模板绑定、模板版本、预览与渲染契约
- 建立内容生命周期与发布契约
- 建立信创与国密基础设施抽象

验收：

- 共享对象与接口边界固定
- 模板管理边界与渲染契约固定
- 生命周期与发布契约可落地
- 安全基础设施方案定型

状态：进行中。栏目管理、模板管理、内容生命周期、发布中心 MVP 已完成首批落地；导航、专题、站点管理员仍待进入实现。

### M3 后台标准版

目标：

- 实现站点工作台、栏目管理、内容中心、媒体中心、专题管理、模板管理、审核中心、发布中心
- 形成后台完整运营闭环

验收：

- 建站 -> 编辑 -> 审核 -> 发布流程可跑通
- 权限模型与业务域一致
- 审计和发布记录可追踪

状态：进行中。内容中心、审核工作区、发布中心 MVP 已落地，但站点治理、专题管理、审计中心仍未完成。

### M4 门户标准版

目标：

- 实现首页、栏目页、详情页、专题页
- 实现静态化整站发布和增量更新
- 实现门户少量动态能力接入

验收：

- CMS 数据可以生成门户站点
- 整站发布与增量发布都可执行
- 页面结构、导航、分页、SEO 元信息正确

状态：规划中。

### M5 正式交付与环境联调

目标：

- 完成源码、脚本、配置、手册交付
- 完成信创环境部署验证
- 完成国密链路验证

验收：

- KingbaseES、TongWeb、Nginx、国产 OS 方案可落地
- 国密传输、签名、数据保护可用
- 可以按交付文档独立部署和回滚

状态：规划中。

## 2. 总体验收口径

项目最终验收必须同时满足四类条件：

- 功能：后台和门户的核心闭环完整可用
- 发布：整站发布、增量发布、回滚和日志可用
- 安全：国密能力、审计能力、基础安全防护可用
- 交付：源码、脚本、配置、运维文档齐全
