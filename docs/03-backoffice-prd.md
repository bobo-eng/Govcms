# GovCMS 后台标准版 PRD

## 1. 目标

后台标准版是 GovCMS 的运营核心，用于完成站点搭建、栏目配置、内容编辑、审核发布、媒体管理和发布运维。

后台标准版不是简单的 CRUD 集合，而是一个完整的工作台，目标是让运营人员能够独立完成“建站 -> 编辑 -> 审核 -> 发布 -> 追溯”的闭环。

## 2. 目标角色

| 角色 | 主要职责 |
| --- | --- |
| 平台管理员 | 管理系统配置、权限、安全策略、审计查询 |
| 站点管理员 | 管理站点、栏目、导航、模板、发布配置 |
| 编辑 | 创建和编辑内容、专题、媒体引用 |
| 审核人员 | 审核内容、驳回内容、确认发布前状态 |
| 运维人员 | 执行发布、查看发布记录、处理回滚 |

## 3. 模块范围

### 3.1 站点工作台

- 管理站点基础信息、域名、导航配置、首页配置
- 管理站点状态、发布状态、最近发布记录
- 作为进入栏目、内容、专题和发布中心的统一入口

### 3.2 栏目管理

- 支持树形栏目结构
- 支持栏目名称、编码、描述、排序、模板、状态配置
- 支持栏目导航与显示规则
- 支持栏目下内容聚合规则

### 3.3 内容中心

- 内容创建、编辑、预览、送审、驳回、重新提交、发布、下线
- 内容必须归属站点与栏目
- 内容支持摘要、正文、作者、来源、封面、标签、置顶等基础字段
- 内容支持媒体资产引用

### 3.4 媒体中心

- 媒体上传、预览、删除、筛选、引用追踪
- 支持图片、PDF、文档等基础媒体类型
- 支持标记媒体被哪些内容或专题引用
- 为发布中心提供受影响页面识别能力

### 3.5 专题管理

- 专题创建、编辑、预览、发布、下线
- 专题可聚合栏目、内容、图片和推荐位
- 首期采用标准专题模板，不做自由拖拽搭建

### 3.6 审核中心

- 首期固定为单级审核流
- 状态流转固定为：`draft -> review -> rejected/published -> offline`
- 审核人员可以查看送审列表、驳回原因、重新送审历史

### 3.7 发布中心

- 支持整站发布
- 支持按内容、栏目、专题触发增量发布
- 支持发布预览、发布记录、发布结果、失败重试、回滚说明

### 3.8 系统配置与审计

- 查看系统安全配置、发布配置、密钥配置引用状态
- 查看操作日志、发布日志、异常日志、审计日志
- 满足信创与国密运维排查要求

## 4. 核心流程

### 4.1 建站流程

站点管理员创建站点后，配置栏目树、导航和模板，再进入内容填充和专题配置。

### 4.2 内容流程

编辑创建内容，完成内容编辑与预览后送审；审核人员通过或驳回；通过后进入发布中心统一发布。

### 4.3 发布流程

发布中心根据触发对象计算受影响页面：

- 单篇内容：详情页、所属栏目页、首页推荐位、搜索索引
- 栏目变更：栏目页、栏目导航、栏目聚合位
- 专题变更：专题页与首页专题位
- 站点变更：整站发布
